Setup IntelliJ projects

In the theory, the developers could develop MyCollab by any IDEs in the market, some of them are Eclipse, Netbeans or IntelliJ. Eclipse used to be our favorite IDE to develop MyCollab, however when MyCollab codebase grew fast then Eclipse is very slow and can not compile the whole projects without endless loop of clean and compile commands. If you want to use Eclipse to customize MyCollab, this link may be helpful http://community.mycollab.com/set-up-eclipse-workspace/, however we do not support for any question relate to the Eclipse IDE.

We strongly recommend IntelliJ, a powerful IDE for Java and Scala language. All MyCollab developers uses IntelliJ as our development tool until we have any further change. If you have any issues to set up MyCollab with IntelliJ and need help, you can ask the question to Github issue https://github.com/esofthead/mycollab/ issues or contact us via MyCollab contact form https://www.mycollab.com/contact/

Clone the latest MyCollab sources from Github

You can use your favorite Git client or git command line to clone MyCollab repository from https://github.com/esofthead/mycollab.git

Download the IntelliJ IDE if you don’t have it yet

Download the latest IntelliJ version at https://www.jetbrains.com/idea/download/. The community version is enough for developing the MyCollab application.

Install IntelliJ Scala plugin

MyCollab uses Scala is the second programming language besides Java, and we will migrate some Java codes to Scala in the upcoming MyCollab versions. Without the Scala plugin, the IntelliJ may inform you there are some classes could not be found.

Go to File > Project Structure, and choose the Scala SDK used in MyCollab poject and modules

Import the Maven project to IntelliJ

Select File > New > Project from existing sources … then choose the folder you checkout the latest version of MyCollab from Github

then choose the import project from Maven external model

IntelliJ will create an empty project names ‘mycollab-deployer-community’. Select File > New > Module from existing sources … and you select the folder you checkout MyCollab from Github again. IntelliJ will import all MyCollab modules to the project as the following screenshot

Then you would like to enable Scala in MyCollab modules. Go to Settings > Plugins and search Scala, then install the Scala plugin. Restart the IntelliJ to make the change is affected.

Then make by selecting Build > Make Project the whole MyCollab modules. The compile process should work without any issue.

Run MyCollab

If you pass all above steps successfully, then this task is simple by create a Java application with main class com.esofthead.mycollab.jetty.GenericServerRunner in the module mycollab-app-community with following settings:

  •  VM Settings:  -Xmx512m -Xms256m -XX:MaxPermSize=128m
  • Program arguments: –port 8080 (you can replace 8080 by any port you want MyCollab web run with)
  • Working directory: the module MyCollab program is running. In this case, it is mycollab-app-community

 

You can see our settings to run the MyCollab as in the screenshot. Any question should be discussed in the project github issues
